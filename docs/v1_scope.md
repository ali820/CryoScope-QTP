# CryoScope-QTP V1 范围定义

## 版本定位

- 版本目标：交付一个面向青藏高原冻土研究的可运行 `MVP`，让研究人员可以在单个界面内完成基础问答、材料上传、规则辅助判断和快速数据体检。
- 目标用户：冻土研究人员、研究生、课题组成员、需要整理观测材料和学术文本的科研人员。
- 发布形态：单机或单实例部署的 `Streamlit` 应用。

## 本版成功标准

- 能在本地按文档完成安装并正常启动。
- 主聊天页在有 API Key 时能调用模型，在无 API Key 时能返回离线提示。
- 用户可以上传 `txt/md/csv/xlsx/pdf/docx` 材料，常见格式失败时不会直接导致页面崩溃。
- 数据解读页能完成基础统计、趋势和相关性摘要。
- 航测 / DEM 诊断页能输出风险等级、主要发现、不确定性和建议动作。
- 规则文件可在页面中查看、编辑并写回本地文件。

## P0 必做范围

- 主聊天页
- 任务模式切换
- 本地知识库检索
- 上传文件解析
- OpenAI / 兼容网关调用
- 无 API Key 时的离线降级提示
- 数据解读页
- 航测 / DEM 诊断页
- 规则库查看与编辑
- 基础部署说明

## P1 应做范围

- 规则库和知识库更新后的重载入口
- 基础的自动回退模型策略
- 最小自动化测试
- 上线前检查清单
- 版本范围和验收文档

## P2 暂不纳入本版

- 多用户登录与权限管理
- 云端数据库或用户会话持久化
- 向量数据库与高召回 RAG
- OCR 扫描版 PDF 识别
- 完整的审计日志与观测指标平台
- 自动化部署流水线
- 复杂图表分析和地图可视化

## 验收标准

- `README` 中的安装和启动步骤可直接复现。
- `tests/` 中的最小回归测试可执行通过。
- 页面导航包含主聊天、数据解读、航测 / DEM 诊断、规则库、部署说明。
- 无效表格上传会给出错误提示，而不是让应用直接中断。
- 自定义网关模式下，快速提示词仍会保留本地知识命中摘要。

## 已知风险

- 当前仍是单文件应用，后续功能扩展会提高维护成本。
- 大模型调用依赖外部服务可用性和网络质量。
- 轻量关键词召回适合原型验证，不适合替代正式知识检索系统。
- 规则驱动判断适合第一轮筛查，不替代正式误差分析和科研结论。

## 发布前门槛

- 手工 smoke test 完成。
- 最小自动化测试通过。
- `rules/` 与 `knowledge/` 已补齐实际业务内容。
- API 配置、部署路径和回滚方案已确认。
