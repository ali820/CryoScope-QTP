# CryoScope-QTP V1 上线检查清单

## 发布前

- 确认本次上线范围与 [docs/v1_scope.md](v1_scope.md) 一致。
- 确认 `requirements.txt` 与实际运行环境一致。
- 确认 `.streamlit/secrets.toml` 或环境变量已正确配置。
- 确认 `rules/` 与 `knowledge/` 已更新为正式内容，而不是占位样例。
- 确认所有本地改动已归档到版本管理工具。

## 测试检查

- 执行 `bash scripts/verify_local.sh`
- 启动应用并人工检查 5 个页面都可打开。
- 无 API Key 时主聊天页能返回离线提示。
- 有 API Key 时主聊天页能完成一次真实问答。
- 上传一个正常 `csv/xlsx`，确认数据解读页能展示摘要和图表。
- 上传一个损坏表格文件，确认页面显示错误信息而不是崩溃。
- 在航测 / DEM 诊断页输入高风险案例，确认输出等级和建议。
- 在规则库页修改一个规则文件并确认能保存。

## 发布执行

- 执行 `bash scripts/run_local.sh`
- 记录本次发布时间、运行目录、模型配置、知识库版本。
- 保留上一版 `rules/`、`knowledge/` 和配置文件备份。
- 首次发布优先在内网、小范围或本机环境验证。

## 回滚方案

- 停止当前进程。
- 恢复上一版 `rules/`、`knowledge/` 和 `.streamlit/secrets.toml`。
- 切回上一版应用代码后重新启动。
- 再次执行最小 smoke test，确认核心页面恢复正常。

## 发布后观察

- 观察模型调用是否经常超时或失败。
- 观察文件上传是否出现解析异常。
- 观察规则库修改后是否能即时生效。
- 收集研究人员反馈，记录下一版需求。
